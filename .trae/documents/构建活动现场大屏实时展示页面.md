这是一个非常好的需求，可以极大地增强现场互动氛围。

**需求分析：**
您希望为活动组织者提供一个“大屏展示模式”（Live Dashboard），主要用于活动现场的大屏幕投影。
该页面需要具备以下特点：
1.  **大字号、高对比度**：适合远距离观看。
2.  **扫码入口**：展示巨大的二维码，方便现场观众随时扫码加入或登录。
3.  **实时数据**：动态显示当前报名人数、已提交选择的人数等关键指标。
4.  **无需登录**（或简化验证）：考虑到是现场投屏，URL 应该是公开的（或者通过活动 Token 访问），但不应暴露敏感信息（如参与者具体名单、联系方式等）。

**实施方案：**

我将创建一个新的公共页面：`/e/[eventNo]/live`。

**1. 页面设计 (UI/UX)：**
*   **布局**：左右分栏或上下分层。
    *   **核心区（左侧/上部）**：超大二维码 + "Scan to Join" 提示。
    *   **数据区（右侧/下部）**：
        *   **参与人数 (Participants)**：大数字展示。
        *   **已提交 (Submitted)**：大数字展示（可选：进度条）。
        *   **倒计时 (Countdown)**：如果活动未开始，显示倒计时；如果进行中，显示已进行时长。
*   **风格**：深色模式（Dark Mode）默认，酷炫的背景动画（如粒子效果），营造高端氛围。

**2. 技术实现：**
*   **路由**：`src/app/[locale]/e/[eventNo]/live/page.tsx`。
*   **数据获取**：
    *   复用 `/api/events/public?eventNo=...` 获取基础信息。
    *   **实时性**：使用 `SWR` 或简单的 `setInterval` 轮询（例如每 5 秒刷新一次），获取最新的人数统计。
    *   为了保护隐私，该页面**只展示统计数据**，不展示具体人名列表。

**3. 入口：**
*   在组织者的活动详情页 (`/my-events/[eventId]`) 的 "Share" 标签页或顶部操作栏中，增加一个 "Open Live Screen" 按钮，点击跳转到该页面。

**计划步骤：**

1.  **创建 Live 页面**：新建 `src/app/[locale]/e/[eventNo]/live/page.tsx`。
2.  **实现实时轮询**：在 Live 页面中集成 API 轮询逻辑，实时更新 `currentParticipants` 和 `submittedChoicesCount`（后者可能需要后端 API 支持返回统计）。
3.  **更新 API**：修改 `/api/events/public`，确保它返回 `submittedChoicesCount`（目前可能只有总人数）。
4.  **添加入口**：在组织者后台增加跳转按钮。
