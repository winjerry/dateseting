# 约会 SaaS 核心整改任务

## 目标

围绕 `报名 -> 支付 -> 建活动 -> 组织者管理 -> 会后匹配` 主闭环完成修复，优先解决数据一致性、安全和业务缺口。

## 执行顺序

### P0：上线阻断

1. 报名页接入真实活动信息与自拍上传
2. 报名接口改为事务化，防重复报名和超卖
3. 修复支付创建失败状态
4. 修复支付回调幂等，避免重复建活动
5. 给状态更新 cron 接口加鉴权

### P1：业务闭环

6. 增加组织者活动更新接口
7. 增加容量管理、支持信息、参与者导出
8. 修复编辑正式活动仍跳支付的问题
9. 修复 participant list / match results 邮件内容与状态回写
10. 给 choices / matches 增加活动时序约束
11. participant 登录时刷新 choice token

### P2：体验与质量

12. 清理核心页面乱码和错误空状态
13. 修正 organizer 详情页 tab 栅格和无效按钮
14. 补充必要环境变量和配置说明

## 验收清单

- 报名页显示真实活动信息，可上传自拍
- 同邮箱不能重复报名同一活动，并发不超卖
- 支付失败不会误标记为完成
- 支付回调重复命中不重复创建活动
- 未授权不能触发 cron
- organizer 可编辑活动、扩容、导出名单、查看支持信息
- participant list 邮件只发 first name + photo
- match results 邮件发送后有状态回写
- 活动未结束前不能提交 choices 或计算 matches
- 登录 choices 前 token 会刷新
- 核心页面无乱码，主要 CTA 可用
